## 1. Purpose

`leaser.rs` implements single-primary fencing for the rustyriver replication loop via an
object-storage distributed lease.  A node must hold a live lease before writing any LTX
data to the remote store; if it cannot acquire or renew the lease, it must stand by
(read-only mode) until the current lease expires and becomes available.

The lease is a tiny JSON file (`lock.json`) written to the replica's object-storage prefix.
All acquire/renew/release operations are guarded by HTTP conditional writes (`If-None-Match:
*` for creation, `If-Match: <etag>` for renewal and deletion), which provide compare-and-swap
semantics over ordinary object storage with no additional primitives.

The `HeartbeatClient` (`heartbeat.go`) is a thin liveness-ping companion that fires an HTTP
GET to a configurable URL at a throttled interval; it is optional infrastructure (empty URL
= no-op) that piggybacks on the renew loop.  It does not interact with the lease protocol
itself.

## 2. Rust API Shape

All async functions target Tokio.  The `Leaser` trait is object-safe and `dyn`-dispatchable
so alternate backends (GCS, Azure) can be added behind feature flags later.

```rust
use std::time::{Duration, Instant};
use serde::{Deserialize, Serialize};

// ── Error additions (add to src/error.rs) ────────────────────────────────────

/// The caller no longer holds the lease (concurrent writer or clock skew).
///
/// Maps to `litestream.ErrLeaseNotHeld` (leaser.go:10).
pub struct LeaseNotHeld;                    // unit struct, impl std::error::Error

/// Another node currently holds a live, unexpired lease.
///
/// Maps to `litestream.LeaseExistsError` (leaser.go:12-22).
pub struct LeaseExistsError {
    /// Identity string of the current holder; may be empty.
    pub owner: String,
    /// Absolute wall-clock expiry of the current holder's lease.
    pub expires_at: std::time::SystemTime,
}

// ── Core lease value type ─────────────────────────────────────────────────────

/// A snapshot of a held lease.
///
/// Maps to `litestream.Lease` (leaser.go:31-36).
/// The `etag` field is NOT serialised to JSON (matches `json:"-"` tag).
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Lease {
    pub generation: i64,
    pub expires_at: std::time::SystemTime,   // serialised as RFC-3339
    #[serde(default, skip_serializing_if = "String::is_empty")]
    pub owner: String,
    #[serde(skip)]                            // matches json:"-"
    pub e_tag: String,
}

impl Lease {
    pub fn is_expired(&self) -> bool;        // time::SystemTime::now() > self.expires_at
    pub fn ttl(&self) -> Option<Duration>;   // None when already expired
}

// ── Leaser trait ──────────────────────────────────────────────────────────────

/// Object-storage lease provider.
///
/// Maps to `litestream.Leaser` interface (leaser.go:24-29).
/// Implemented as a Rust async trait (requires `async-trait` crate, already in
/// Cargo.toml, or use RPITIT on nightly; use `async-trait` for MSRV 1.90).
#[async_trait::async_trait]
pub trait Leaser: Send + Sync {
    /// Short identifier for the backend ("s3", "file", …).
    fn leaser_type(&self) -> &str;

    /// Attempt to acquire the lease. Returns `Err(LeaseExistsError)` when
    /// a live unexpired lease is held by another node.
    async fn acquire_lease(&self, ctx: ()) -> crate::Result<Lease>;

    /// Renew a held lease.  Returns `Err(LeaseNotHeld)` when the
    /// conditional write fails (ETag mismatch — another node took over).
    async fn renew_lease(&self, lease: &Lease) -> crate::Result<Lease>;

    /// Release a held lease. Returns `Err(LeaseNotHeld)` on ETag mismatch.
    async fn release_lease(&self, lease: &Lease) -> crate::Result<()>;
}

// ── S3 implementation ─────────────────────────────────────────────────────────

/// S3-backed lease using `object_store`.
///
/// Maps to `s3.Leaser` (s3/leaser.go:42-50).
pub struct S3Leaser {
    store: std::sync::Arc<dyn object_store::ObjectStore>,
    bucket: String,
    path: String,      // prefix; lock key = path + "/lock.json"  (s3/leaser.go:83-88)
    ttl: Duration,     // default 30 s  (s3/leaser.go:22)
    owner: String,     // default hostname:pid  (s3/leaser.go:52-65)
}

impl S3Leaser {
    pub fn new(store: std::sync::Arc<dyn object_store::ObjectStore>) -> Self;
    pub fn with_bucket(self, bucket: impl Into<String>) -> Self;
    pub fn with_path(self, path: impl Into<String>) -> Self;
    pub fn with_ttl(self, ttl: Duration) -> Self;
    pub fn with_owner(self, owner: impl Into<String>) -> Self;

    fn lock_key(&self) -> object_store::path::Path; // "path/lock.json" or "lock.json"

    async fn read_lease(&self) -> crate::Result<Option<(Lease, String)>>;
    // returns (Lease, e_tag) or None when object is absent

    async fn write_lease(
        &self,
        lease: &Lease,
        current_e_tag: Option<&str>,   // None  -> If-None-Match: * (PutMode::Create)
                                       // Some  -> If-Match: etag  (PutMode::Update)
    ) -> crate::Result<String>;        // returns new e_tag
}

#[async_trait::async_trait]
impl Leaser for S3Leaser { /* … */ }
```

## 3. Format-Layer Integration Points

T15 sits above the object-store I/O layer (T7) and does NOT directly consume `TXID`,
`Pos`, or LTX frame data.  It is a pure fencing primitive.  What it does consume:

| Item | Where it lives | What T15 needs |
|---|---|---|
| `crate::Error` | `src/error.rs` | Must add `LeaseNotHeld` and `LeaseExistsError` variants (or keep them as separate concrete types convertible via `From`). The existing `Error::Other` can serve as a placeholder until dedicated variants land. |
| `crate::Result<T>` | `src/error.rs` | Used as the return type throughout. Already exists. |
| `Arc<dyn ObjectStore>` | `object_store` crate (already in Cargo.toml) | T15 directly calls `store.get()`, `store.put_opts()`, and `store.delete()`. These are available in `object_store 0.11.2` (pinned in Cargo.lock). |
| `PutMode::Create` | `object_store::PutMode` | Maps to `IfNoneMatch: *` (s3/leaser.go:246-248). Available in 0.11.2. |
| `PutMode::Update(UpdateVersion)` | `object_store::PutMode` | Maps to `IfMatch: etag` (s3/leaser.go:249). `UpdateVersion { e_tag: Some(etag), version: None }`. Available in 0.11.2. |
| `object_store::Error::Precondition` | `object_store` crate | Returned when CAS fails; maps to Go's `PreconditionFailed` / HTTP 412. |
| `object_store::Error::NotFound` | `object_store` crate | Returned on absent key; maps to Go's `isNotExists` / `isNotFoundError`. |

**Critical gap — conditional DELETE:** The Go implementation uses `DeleteObject` with
`IfMatch` (s3/leaser.go:180-183) to make release atomic.  The `object_store 0.11`
`delete(&path)` signature takes no options and has **no conditional-delete variant** (see
lib.rs:660).  The porter must choose one of:
- Implement release as a conditional PUT of a sentinel/expired lease record followed by a
  best-effort unconditional delete (slight semantic drift — acceptable if generation is
  incremented on next acquire).
- Open-code the S3 HTTP call directly via the `reqwest` or `aws-sdk-s3` crate for the
  delete path, accepting that this breaks the `object_store` abstraction for one operation.
- Accept a slightly weaker release guarantee: `release_lease` does an unconditional delete
  and returns `LeaseNotHeld` if a subsequent read shows a different owner (read-then-delete
  pattern, TOCTOU window of ~1 RTT).
Log the chosen approach in `OPEN_QUESTIONS.md` (AGENTS.md rule 7).

T7 (`client/object_store.rs`) must be green before T15 can be integrated, because T15's
test helpers spin up an `S3Leaser` against an `AmazonS3` / MinIO-backed `ObjectStore`.

## 5. Footguns & Correctness-Critical Spots

**CAS fencing — the heart of this module.**

1. **If-None-Match: * semantics** (s3/leaser.go:246-248). When no existing lease is found,
   `writeLease` passes `etag = ""` and the PUT uses `IfNoneMatch: *`.  In `object_store`,
   use `PutMode::Create`.  If two nodes race and both see 404, only one CREATE succeeds;
   the loser gets `Error::AlreadyExists` — map this to `LeaseExistsError`.

2. **If-Match on renewal** (s3/leaser.go:249). Pass `PutMode::Update(UpdateVersion { e_tag:
   Some(lease.e_tag.clone()), version: None })`.  A 412 (`Error::Precondition`) means
   another node wrote to the lock key since this node last saw it — map to `LeaseNotHeld`.

3. **412 on acquire -> re-read loop** (s3/leaser.go:115-125). After a failed CAS on
   acquire, the Go code does a second GET to populate `LeaseExistsError` with the winner's
   identity.  Do the same re-read in Rust; do NOT surface a blank `LeaseExistsError`.

4. **ETag round-trip fidelity.** S3 returns ETags wrapped in double-quotes (`"etag-1"`).
   The Go code passes them verbatim (leaser_test.go:27, 97).  `object_store` preserves them
   the same way.  Do NOT strip or re-add quotes; store and compare the raw string from
   `PutResult.e_tag` / `ObjectMeta.e_tag`.

5. **Conditional DELETE gap** (detailed in §3).  This is the biggest correctness risk.
   Pick one strategy, document it, and add a comment `// DEVIATION from Go: …` in the
   source.  The read-then-delete TOCTOU approach is safest with the current `object_store`
   API because it avoids a dependency on raw HTTP.

6. **Generation counter.** Must be monotonically increasing: new generation =
   `existing.generation + 1` (s3/leaser.go:103-107), or `1` when no prior lease exists.
   Renewal keeps the same generation (s3/leaser.go:146-148).  Any bug here breaks the
   failover detector that the application layer uses to detect split-brain.

7. **TTL clock.** `expires_at = now() + ttl` is computed at write time, not at read time.
   Clocks on two nodes may differ.  The default TTL is 30 s (s3/leaser.go:22); use
   `std::time::SystemTime::now()` (not `Instant`) because this value is serialised to JSON
   and compared cross-node.

8. **Serialisation format.** The JSON body is the `Lease` struct with `generation` (i64),
   `expires_at` (RFC-3339 string), and optional `owner`.  The `e_tag` field is NOT
   serialised (`json:"-"` in Go → `#[serde(skip)]` in Rust).  Use `chrono` or `humantime`
   for RFC-3339 `SystemTime` serialisation; do NOT use `serde_json`'s default
   `SystemTime` serialiser (it uses epoch seconds, not RFC-3339).

9. **`HeartbeatClient` interval floor.** `NewHeartbeatClient` enforces
   `interval >= MinHeartbeatInterval` (1 minute) regardless of the caller's value
   (heartbeat.go:29-31).  Replicate this clamp in the Rust constructor.

10. **Object-store `delete` on a non-existent key.** The `object_store` spec says "the
    result may be an error or a success" for absent keys (lib.rs:673).  The Go code
    explicitly treats 404 on DELETE as `ErrLeaseAlreadyReleased` (s3/leaser.go:186-188).
    Always check for `NotFound` before returning success on release.

11. **Concurrency within a single process.** The Go `Leaser` has no internal mutex; it
    relies on the CAS protocol for cross-process safety.  Within one Rust binary, if
    multiple tasks call `acquire_lease` concurrently, the CAS still serialises them
    correctly via the object store, but only one caller's `Lease` handle will be valid.
    The application layer is responsible for holding exactly one `Lease` at a time; the
    leaser itself need not have a `Mutex`.

## 7. Size & Risk

**LOC estimate:** ~350-420 Rust LOC (the Go is 305+44+84 = 433 LOC total, plus the test
file at 603 LOC).  The Rust source will be somewhat shorter due to trait-based dispatch and
the absence of the SDK boilerplate that `object_store` already absorbs, but the test file
will be comparable in size.

**Risk: MED**

The fencing protocol itself is straightforward (three operations, well-covered by the Go
tests), but two issues raise the risk above LOW:

- The **conditional-delete gap** in `object_store 0.11` requires a deliberate design choice
  that slightly deviates from the Go reference; the chosen approach must be correct under
  concurrent writers.
- **RFC-3339 SystemTime serialisation** has no single obvious Rust crate equivalent to
  Go's `time.RFC3339`; the wrong choice will silently break cross-version JSON
  compatibility.

Both issues are well-defined and have clear solutions; neither requires fundamental
rearchitecting.
